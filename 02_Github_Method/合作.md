## 一、加入别人的库后如何在本地编辑

### 法一：VS Code

1. `Ctrl+Shift+P` 打开命令面板，输入 `Git: Clone`
2. 输入仓库地址：`https://github.com/wsgbb/price-compare-agent` （*但是神秘的是我好像没有在末尾加git，而这次上传的时候就不用调端口了！一会去试一下*）
3. 选择本地文件夹

### 法二：命令行

```bash
# 先 cd 到你想存放项目的父目录
cd ~/Desktop

# 克隆仓库（把下面的地址换成你的）
git clone https://github.com/用户名/仓库名.git

# 克隆完成后，用 VSCode 打开这个文件夹
cd 仓库名
code .
```

## 二、如果本地已经有项目了，如何上传到仓库

1. 在 VSCode 中初始化

    ```bash
    # 打开终端（Terminal），确保你在项目文件夹根目录
    git init
    ```

    或者在VSCode里：
    - 打开文件夹后，按`Ctrl+Shift+P`
    - 输入 `Git: Initialize Repository`
    - 选择当前文件夹

2. 连接到远程 Github 仓库

   ```bash
    # 添加远程仓库地址
    git remote add origin https://github.com/用户名/仓库名.git

    # 推送到 GitHub
    git branch -M main
    git push -u origin main
   ```

## 三、写代码

### 每次写代码前

```bash
git switch main
git pull
```

在 `main` 分支上开发  
`pull` 从仓库下载**最新的代码**，并自动合并到当前分支

> 如果pull的时候发现队友有新内容了，跳出来一个文档，直接关掉即可，默认合并方式

### 每次写代码后

```bash
git status
git diff
git add .
git commit -m "feat: 简短说明"
git pull
git push
```

#### `status`

查看当前工作区哪些文件被修改了、哪些是新增的、哪些被删除了

#### `diff`

查看具体修改的内容（每一行代码的变化）。

操作方式如下：
| 按键 | 作用 |
| --- | --- |
| `空格` | 下一页 |
| `回车` | 下一行 |
| `b` | 上一页 |
| `q` | 退出 |
| `/关键词` | 搜索 |
| `h` | 查看帮助 |

#### `add .`

把当前目录下**所有修改过的文件**添加到“暂存区”（相当于打包准备提交）

#### `commit -m "feat: 简短说明"`

把暂存区的修改“正式存为一次版本记录”，并加上说明。

#### `pull` (**关键一步**)

在你 commit 之后、push 之前，再拉一次**最新代码**。  
改代码期间，队友可能又推送了新内容。先拉下来，如果没冲突就能直接合并，再推上去。

#### `push`

把你本地提交的版本记录上传到 GitHub，让队友能看到。
