# 三角形与集合

> 出自2025年4月左右，高三时瞎编来玩的
> 参考那种集合题，为集合赋予某种特殊的条件，再通过该条件研究集合的属性
> 高中时觉得最好玩的就是集合了，虽然高一时难题基本不会哈哈哈

## 题目

> $已知集合A=\{1,2,...,1000\}，以及其子集 S \subseteq A 。子集S满足条件：对于\forall a,b,c \in S，且a,b,c不相等，a，b，c能作为三角形的三条边的长度，构成一个三角形。$
> $现在求S的元素个数的最大值M = max\{ card(S) \}$

## 答案

$501$

## 解法

### 速解

$从1000，999，998开始，往前加元素，加到500加不了了，共501个元素。$

### 严谨解

$对于任取的a, b, c, 我们不妨设a < b < c$
$而对于S，不妨设a, b是S的最小和第二小的元素，接下来我们来通过a，b表示S的元素个数$
$将自然语言翻译成数学语言：$ $$\forall a<b<c \in S, a\neq b, a\neq c, b\neq c, 使得 c<a+b$$ $此时注意到已知a，b下，c的取值范围是b+1,b+2,...,b+a-1$
$此时我们可以验证，对于集合S=\{a,b,b+1,b+2,...,b+a-1\}满足上述条件！$
$接下来我们验证S不能再添加任何其他元素了。对于\forall x \notin S，我们可以分类讨论。$
$1. x < a: 此时一定有b<b+x<b+a，故b+x \in S，此时若取x,b,b+x，构不成三角形，故错$
$2. a < x < b: 有x+a<b+a，\exist p \in S ,b<p<b+a 使x+a \leq p 此时若取x,b,p，不是三角形，错$
$3. b < x: 若取a,b,x，显然x不满足上面的c的条件，故错误$
$因此我们得到了用a，b表示的S的一般表达式！以及其元素个数表达式：$ $$card(S) = ((b+a-1)-b+1)+1 = a+1$$ $接下来我们只需寻找a的最大值即可。$
$S的前提条件是A的子集，即S的上界sup(S)=a+b-1\leq1000，下界inf(S)=a\geq1$
$此时a满足：$ $$1\leq a < b ,a+b \leq 1001$$ $要使a大的同时也要保证a+b<=1001，则b得尽可能的小，即a+1，此时a = 500。$
$要证明a就是最大的，不妨证不存在比500更大的a：$
$因为a'\geq 501,b'>a'即b' \geq a'+1 \geq 502,则a'+b' \geq 1003，不满足a+b\leq 1001$
$得证，在a=500时，S的元素个数最大，为501。$

:::info 补充1
如果想要正着严谨地证明b=a+1的话，可以参考：
$对于a,b，满足1\leq a < b ,a+b = 1001，且b>a+1，a，b一定是一奇一偶，其间距为偶数$
$不妨使a+1，b-1，此时上限不超过的情况下，a增大了，以此类推，最终a，b相遇。$
$而对于a+b<1001，同上易证明此时的a一定小于上述a，故上述的a一定是最大的。$
:::
:::info 补充2
S的上界的确可能是1000，而a+b-1大于1000，此时S的元素会小于a+1
易证越界的S的元素个数一定小于501（下界增大上界不变，元素必然减少）
:::
