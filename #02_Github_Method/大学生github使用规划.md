## 大一

主要是将学习过程记录下来、上传，一个作用是备份，另一个作用是让别人能看到自己实打实的努力。

学命令。

推荐学习资料：

- [ProGit](https://git-scm.com/book/en/v2)
- 廖雪峰的Git教程

## 大二

逛Github：搜索资源、按星级排序
学习资料：Awesome系列
找到有趣的，先git clone下来，看readme，再拆解程序

## 大三

> 从围观到搭把手

参与！
第一步贡献可以很简单：

- 帮忙修typo
- 补readme
- 翻译一段文档
- 提issue（遇到bug）

很多大型项目有 `good first issue` 的标签，为新手准备，像是改错别字什么的。

## 大四

打造技术名片

1. 把好的项目、玩意、报告等整理好，放到github上
2. README写好
   1. 这个项目是干嘛的（一两句）
   2. 有效果图或者动图展示吗？
   3. 用了哪些技术？
   4. 怎么运行（提供清晰的步骤）
3. 保持 “绿格子” 好看
4. pin几个得意之作（放自己主页首页）
